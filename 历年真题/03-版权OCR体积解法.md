# 版权 · OCR · 体积 — 三道坎怎么解

> 日期：2026-07-24  
> 背景：ForestDeer 公开真题库「试扫 → 笔记 → 私有 GitHub」已通，但**全量搬 PDF** 被三件事卡住。  
> 本文给出**可落地**的解法（已装 RapidOCR 做中文对比验收）。  
> 关联：[[02-ForestDeer试扫记录]] · [[01-GitHub类似库对照]] · [[00-资料来源与使用说明]] · [[高等数学/_ocr_compare_math2020_p1-2]]

---

## 一句话结论

| 坎 | **正确解法** | **错误解法** |
|:---|:---|:---|
| **版权** | 只推 **Markdown 讲解/结构/范文**；PDF 永远本地缓存 | 把 1GB 原卷推进 GitHub（哪怕私有也有版权风险） |
| **OCR** | **分流引擎**：文字层 / 中文扫描 / 公式 分开 | 指望 Tesseract 一次搞定高数公式 |
| **体积** | Git 只装笔记；缓存盘外挂 + `.gitignore` 拦二进制 | Git LFS 塞整仓真题 PDF（贵、慢、仍有版权问题） |

**功能完善后再「适当加入」的含义**  
= 扩 **年/科的 Markdown 笔记**，**不是**扩 Git 里的 PDF 体积。

## 坎 1：版权（不能「解决成合法整卷开源」）

### 现实

- 广东省考试院真题 + 答案的**全文复制/公开传播**受著作权法限制。  
- GitHub 上几乎没有「干净合法的官方原卷全文开源库」——正常。  
- ForestDeer 是**民间资料合集**，有水印站点、机构痕迹；适合当**索引与对照源**，不是法源。

### 本库边界（硬规则）

| 可以进 Git（私有笔记库） | 禁止进 Git |
|:---|:---|
| 题型结构、分值、年份索引 | 完整官方/扫描 PDF、DOCX 原卷二进制 |
| 考点讲解、得分点、默写清单 | 1:1 整卷影印式粘贴当「泄题包」传播 |
| 写作范文（精析可默写版，经校对） | 把对方 1GB 仓整 fork 进本库 |
| 与回忆卷/纸质册的**差异提示** | 对外二次分发 PDF |

### 三级来源（复习用）

详见 [[00-资料来源与使用说明]]：

1. **A** 考试院公开政策/大纲  
2. **B** 真题题型与考点复现（回忆 + 正版册 + 公开精析）  
3. **C** 同型演练题  

**最终题干/答案以你手里的正版广东真题册为准。**

### 若要「更多原卷」怎么存

```
D:/专升本/_pdf_cache/forestdeer/     ← 本地缓存，永不 git add
D:/专升本/专升本/资料/*真题原卷/      ← 已在 .gitignore（*.pdf / *.docx）
本库 历年真题/**/*.md                ← 唯一推送物
```

**不要用 Git LFS 把真题 PDF 塞进仓库**——LFS 只解决「大文件传得动」，**不解决版权**；私有仓被共享、泄漏或换号后仍有风险。

## 坎 2：OCR（用「分流」而不是「换一个神器」）

### 本机实测（2026-07-24 · 高数 2020 扫描 PDF · 250dpi · 第 1–2 页）

| 引擎 | 中文卷头/结构 | 公式 | 耗时/页（约） | 角色 |
|:---|:---|:---|:---|:---|
| **RapidOCR**（`rapidocr-onnxruntime`，已装） | **明显更好**，题号、注意事项可读 | 仍糊（lim/上下标不可靠） | ~6–7s CPU | **扫描中文默认** |
| **Tesseract 5.5** `chi_sim+eng` | 能认出但错字多 | 很差 | 相近 | 备用 / 纯英文 |
| **文字层 PDF**（英语精析） | — | — | 秒级 | **优先 `pdftotext` / mcp-pdf `read_pdf_text`**，根本不 OCR |
| **DOCX**（计算机 unlocked） | — | — | 秒级 | **python-docx**，不 OCR |

完整对照摘录：[[高等数学/_ocr_compare_math2020_p1-2]]  
本地原图/对照：`D:/专升本/_pdf_cache/forestdeer/_txt/compare_rapid_vs_tess.md`

### 推荐流水线（分流）

```
                    ┌─ 有文字层？ ─YES→ pdftotext / read_pdf_text → 校对 → md
PDF/DOCX 进来 ─→ 判类型
                    ├─ DOCX？ ─YES→ python-docx → 校对 → md
                    └─ 扫描件？ ─YES→ 分页渲染(200–300dpi)
                                      → RapidOCR（中文）
                                      → 结构题号入库
                                      → 公式题：人工对着纸质/本库已有年.md 写解法
                                         （可选下一阶段：公式裁剪 + pix2text/MFR）
```

### 为什么公式还是难过关？

- 通用 OCR 按「文字行」切，**数学上下标、分式、积分号**不是它的强项。  
- 业界正经路线是 **「文本 OCR + 公式专用模型」**（如 pix2text / pix2tex-MFR），不是把 Tesseract 调参到 100%。  
- **专升本策略**：OCR 只负责 **卷结构 + 题号 + 题干汉字方向**；计算题仍以本库已有 `高等数学/年.md` 与正版册为准。

## 坎 3：体积（笔记库 ≠ 资料盘）

### 数字感

| 东西 | 量级 | 该放哪 |
|:---|:---|:---|
| ForestDeer 整仓 | ~1GB+ | **不要**进本库 Git |
| 单年英语精析 PDF | ~400KB | 本地 `_pdf_cache` |
| 政治大讲义 | 80MB+ | 本地；按需再碰 |
| 一本 md 真题笔记 | 几 KB～几十 KB | **GitHub** |
| 本库全部 md | 通常 < 几十 MB | GitHub 舒服 |

### 工程做法

1. **`.gitignore` 已拦**：`*.pdf`、`资料/**/*.docx` 等（见仓库根）。  
2. **缓存目录在仓库外**：`D:/专升本/_pdf_cache/`（专升本盘上，不在 `专升本/专升本` Git 根里）。  
3. **按需下载**：只 curl 小体积、有文字层的文件试扫；大包只记链接。  
4. **Git LFS**：**不推荐**用于真题 PDF（版权 + 配额）；若将来要存**自有**小截图/自画公式图，再考虑 LFS 且限扩展名。  
5. **GitHub Release / 网盘**：若必须备份 PDF，用**私人网盘或加密压缩包**，与笔记仓分离。

### 「适当全加入」的正确姿势

| 阶段 | 做什么 | 体积影响 |
|:---|:---|:---|
| 已完成 | 英 2022–24 写作合并；计算机试扫+差异表；高数结构 OCR | 仅 md |
| 下一优先 | 文字层/DOCX 继续扩年 → 合并进正式 `年.md` | 仅 md |
| 扫描政治/高数 | 分页 RapidOCR → **结构入库**，题目讲解手写/对照本库 | 仅 md |
| 永不做 | `git add` ForestDeer 全量 PDF | 炸仓 |

## 参考（公开技术资料，非法律意见）

- PaddleOCR vs Tesseract：深度学习流水线在复杂/中文场景通常更稳；干净印刷件 Tesseract 仍可用。  
- RapidOCR：ONNXRuntime 轻量部署，适合本机 CPU。  
- Git LFS：解决大二进制版本管理，**不是**教育资料版权保险箱。
