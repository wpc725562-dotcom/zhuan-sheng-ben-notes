## 三、公开库 vs 本库：分工建议

```
┌─────────────────────────────┐
│  正版纸质真题汇编（一手）    │  ← 最终对答案、对题干
└─────────────┬───────────────┘
              │
    ┌─────────┴─────────┐
    ▼                   ▼
 公开 PDF 库           本库 Markdown
 ForestDeerDev…        历年真题/学科/年.md
 找原卷、打印          每天默写、得分逻辑
              │
              ▼
        你的本地 `资料/*真题原卷/`
        （已上传的 PDF/DOC，默认可不 push 大文件）
```

| 你想做的事 | 去哪 |
|:---|:---|
| 默写英语作文 / 通知 | 本库 `公共英语` + 作文模板笔记 |
| 看 2021–2024 计算机原题文件 | 公开库「计算机」docx **或** 本地 `资料/计算机真题原卷/` |
| 政治大题背诵册 | 公开库政治目录有大量 PDF；本库练「框架 + 要点」 |
| 志愿 / 投档线 | 公开库 `志愿与投档数据/`（本库未收录） |
| 同步到手机/另一台电脑 | 本库 GitHub 私有库 `git push` |

## 四、检索时发现的边界（忍者实话）

1. **GitHub 几乎没有「干净、完整、纯官方原卷 + 官方答案」的合法开源库**——正常。官方卷不是开源软件。  
2. 最大的对口公开库是 **PDF 资料堆**，适合「有文件可下」，不适合「Obsidian 双向链接 + 按年讲解」。  
3. 本库之前写的「不镜像整卷」仍然正确；**对照公开库 ≠ 把对方 PDF 全部 commit 进私有库**。  
4. 若以后要「加深」某一科：  
   - 优先把你已有本地原卷 OCR/摘题写进对应 `YYYY.md`；  
   - 公开库只作「缺哪年文件」的线索。
